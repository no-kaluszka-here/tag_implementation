UA Event Tracking

Analytics event should not be considered exclusively as the way to define a clicked element; however, in most cases it is so. With event tracking you can trace any action that happened on the webpage, so let's take a look at few examples.

How do we send events?


To put it simply we can use an HTML event, which is responsible for firing when some action is happenning on the webpage. Lets take a look at the anatomy of a simple event which is being sent when someon clicks on the button.

<button onclick="gtag('event','this-is-click',{'event_category':'some_category'})">

  • <button> marked with red represents an opening tag of the button element. In HTML elements have opening < > and closing </ > tags. Everything between them is considered a content of the element. A simple button element is written as <button> </button>, form - <form> </form>, a clickable link - <a> </a>

  • onclick= marked with pink is an HTML event. Those are representatives of different actions that may occure on the webpage. The names of HTML events are quite intuitive, so "onclick" literally means "when the element is clicked", "onsubmit" is another HTML event - "when the form is submitted", "oncopy" fires "when the user copies the content of the element". Event keyword is put in the opening tag of the element and is followed by the equals sign "=", so everything that follows tells us what is triggered.

  • "gtag('event','this-is-click',{'event_category':'some_category'})" marked with green is an actual event that we are sending to Google Analytics. "gtag" is a function and it's sending arguments to Analytics. The first argument tells what kind of information we send - 'event', the second argument is the name of the event (referred to as "action"), the third is a set of event parameters: category, value, label.


As you may notice, we insert our "event code" inside the **opening tag of the button <button>. We start implementation with initializing an HTML event - something that will tell our code when to fire. In this case it's onclick, meaning that the code will fire when the element is clicked. By putting the equals sign we tell what exactly should happen when the element is clicked. As in out example, when the buttin is clicked we call a fucntion "gtag". Next the function "gtag" sends a set of parameters, first one being 'event', so Analytics knows that we are sending an event, than the name of the event - you can call it whatever you like, and finally - the rest of the parameters (on Developers HC you will find them as category, label and value).

**Normally you will see HTML elements having opening <> and closing </> tags. Everything between them is considered a part of the element.
E.G. <button> This is my button </button> - in this case <button> is an opening tag, </button> is a closing tag and "This is my button" is the text that you will see in the button.

Tips

Definition


Onclick event occurs when the user clicks on an element.

Onsubmit event occurs when a form is submitted.

Onload event occurs when an object has been loaded.

Onplay and onpause events occur when the audio/video has been played/paused accordingly

Action


- it could be a button

This will change your life - or a link




When the body (contents) of this page loaded, it triggered an event


Code


<button onclick="gtag('event','click',{'event_category':'button'})">

<a href="https://youtu.be/dQw4w9WgXcQ" onclick="gtag('event','click',{'event_category':'link'})">

<form onsubmit="gtag('event','form_submission',{'event_category':'form'})">

<body onload="gtag('event','page_load',{'event_category':'page'})">

<audio controls onplay="gtag('event','play',{'event_category':'audio'})" onpause="gtag('event','pause',{'event_category':'audio'})">

Using the examples above you can try creating your own events to practice.
