UA Event Tracking

Analytics event (either UA or GA4) should not be considered exclusively as the way to define a clicked element; however in most cases it is so. With event tracking you can trace any action that happened on the web-page, so let's take a look at few examples.

How do we send events?


To put it simply we can use an HTML event which is responsible for firing when some action is happenning on the webpage. Lets take a look at the atnomy of a simple event which is being sent when someon clicks on the button.
<button onclick="gtag('event','this-is-click',{'event_category':'some_category'})"> As you may notice, we insert our "event code" inside the *opening tag of the button the element is clicked and tell - when this element is clicked you should call a fucntion "gtag". Next the function "gtag" sends a set of parameters, first one being 'event', so Analytics knows that we are sending an event, than the name of the event - you can call it whatever you like, and finally - the rest of the parameters (on HC you will find them as category, label and value).

*Normally you will see HTML elements having opening <> and closing </> tags. Everything between them is considered a part of the element.

E.G. <button> This is my button </button> - in this case <button> is an opening tag, </button> is a closing tag and "This is my button" is the text that you will see in the button.

Definition


Onclick event occurs when the user clicks on an element.

Onsubmit event occurs when a form is submitted.

Onload event occurs when an object has been loaded.

Onplay and onpause events occur when the audio/video has been started/paused accordingly

Action






When this page loaded, it triggered an event


Code


<button onclick="gtag('event','click',{'event_category':'some_category'})">

<form onsubmit="gtag('event','form_submission',{'event_category':'some_category'})">

<body onload="gtag('event','page_load',{'event_category':'some_category'})">
